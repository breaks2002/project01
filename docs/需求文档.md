# 需求文档：Web版ValQ - 驱动型业务规划工具（增强版）

## 1. 项目概述

### 1.1 项目背景

开发一个基于Web的驱动型业务规划（Driver-Based Planning）工具，对标 Power BI 生态中的 **ValQ 3.0** 产品核心功能。该工具将允许业务分析师、财务规划人员快速构建 KPI 驱动模型，进行假设情景分析（What-if Analysis），并可视化展现不同决策带来的业务影响。

### 1.2 产品定位

- **目标用户**：财务分析师（FP&A）、供应链规划人员、销售运营、中小型企业决策者
- **典型应用场景**（参考 ValQ 官网）：
  - 每周销售预测 (Weekly sales forecasting)
  - 滚动预测 (Rolling forecasts)
  - 利润表假设分析 (What-if P&L reporting)
  - 财务预算编制 (Financial budgeting)
  - 三张报表联动模型 (3-Statement financial model)
  - 新产品发布规划 (New product launch planning)
  - 方差报告 (Variance reporting)
  - 运营规划 (Operations planning)
  - 人力规划 (Headcount planning)
  - 战略规划与情景规划 (Strategic planning & Scenario planning)
  - 产能规划（如案例研究中的疫苗生产能力评估）
- **技术形态**：纯前端 Web 应用（HTML/JavaScript/CSS），可本地运行或部署为静态网站，未来可扩展与后端/数据库交互

### 1.3 核心价值（对标 ValQ 用户评价提炼）

- **速度**：复杂计算的实时反馈，“对变化的影响一目了然”
- **可视化**：将 Excel 中容易被忽视的思考过程优雅地呈现出来
- **替代 Excel 疲劳**：无需在漂亮的仪表板和复杂的 Excel 模型之间来回切换
- **直观**：即使面对大量变量，也能在屏幕上清晰组织和洞察

## 2. 功能需求

### 2.1 核心建模能力（模型设计器）

#### 2.1.1 KPI节点定义

| 功能          | 描述                                                         | 优先级 |
| :------------ | :----------------------------------------------------------- | :----- |
| 节点创建      | 用户可以在画布上创建新的 KPI 节点                            | P0     |
| 节点类型      | 支持三种节点类型：**驱动因子**（底层输入）、**计算指标**（由公式定义）、**汇总指标**（聚合子节点） | P0     |
| 节点属性      | 每个节点可配置：名称、单位（如￥、%、件）、默认值、最小值/最大值（用于滑块范围）、数据格式（如小数点位数、千分位分隔符） | P0     |
| 公式编辑器    | 支持类似 Excel 的公式语法（加减乘除、括号、引用其他节点），并支持常用函数（SUM、AVERAGE、IF 等） | P0     |
| 节点删除/复制 | 支持删除已有节点（带二次确认）、复制节点及其公式结构         | P1     |

#### 2.1.2 驱动树构建

| 功能       | 描述                                                         | 优先级 |
| :--------- | :----------------------------------------------------------- | :----- |
| 父子关系   | 支持拖拽连线或通过公式引用建立节点间的依赖关系（如：收入 ← 流量、转化率、客单价） | P0     |
| 自动布局   | 树状结构自动排列，清晰展示驱动层级，避免连线交叉混乱         | P1     |
| 手动调整   | 允许用户手动拖动节点位置（画布式布局）                       | P2     |
| 缩放与平移 | 支持画布的缩放和平移，适应大型模型（如 100+ 节点）           | P1     |

#### 2.1.3 公式语言规范

支持类似 Excel 的简单公式语法，并逐步扩展：

- **基础运算**：`收入 = 流量 * 转化率 * 客单价`
- **复合运算**：`净利润 = 收入 - 成本`
- **比率计算**：`毛利率 = (收入 - 成本) / 收入`
- **时间计算**：`增长率 = (今年 - 去年) / 去年`
- **函数**（P1）：`SUM(节点1, 节点2, ...)`，`AVG()`, `IF(条件, 真值, 假值)`
- **引用**：通过节点名称或 ID 引用

### 2.2 假设情景分析（What-if模拟）

#### 2.2.1 参数调整

| 功能     | 描述                                     | 优先级 |
| :------- | :--------------------------------------- | :----- |
| 滑块控制 | 每个驱动因子节点提供滑块，可动态调整数值 | P0     |
| 数值输入 | 支持直接输入精确数值                     | P0     |
| 调整步长 | 可配置滑块调整步长（如 ±1%、±5%）        | P1     |
| 批量调整 | 允许按百分比批量调整一组驱动因子         | P2     |

#### 2.2.2 实时计算

| 功能         | 描述                                                         | 优先级 |
| :----------- | :----------------------------------------------------------- | :----- |
| 依赖传播     | 任一节点数值变化，所有依赖它的上游节点自动重新计算（响应式依赖图） | P0     |
| 计算引擎     | 基于有向无环图（DAG）的拓扑排序，确保计算顺序正确，无冗余计算 | P0     |
| 环形依赖检测 | 检测并提示公式中的循环引用                                   | P1     |

#### 2.2.3 视觉反馈（用户评价中多次提到的关键点）

| 功能       | 描述                                                         | 优先级 |
| :--------- | :----------------------------------------------------------- | :----- |
| 变化标识   | 节点数值变化时，用**绿色（增加）/红色（减少）**高亮显示数值或背景 | P0     |
| 变化百分比 | 在节点旁显示相对于基准值的变化百分比                         | P1     |
| 影响路径   | 高亮显示受当前调整影响的整个链条，帮助理解传导路径           | P2     |
| 差异箭头   | 显示上升/下降箭头图标，强化视觉感知                          | P1     |

### 2.3 情景管理（Scenario Planning）

#### 2.3.1 多方案创建与对比

| 功能       | 描述                                                         | 优先级 |
| :--------- | :----------------------------------------------------------- | :----- |
| 保存情景   | 将当前所有节点数值保存为一个命名情景（如：保守预测、乐观预测） | P1     |
| 加载情景   | 一键加载已保存的情景                                         | P1     |
| 情景对比   | 并排显示两个情景的数值对比，或用表格展示差异，类似方差报告   | P1     |
| 基准线设置 | 将某一情景设为基准，后续调整显示相对于基准的变化             | P1     |

#### 2.3.2 数据导入导出（与 Excel 工作流衔接）

| 功能                          | 描述                                                         | 优先级 |
| :---------------------------- | :----------------------------------------------------------- | :----- |
| 导出JSON                      | 将完整模型（节点定义+公式+情景数据）导出为 JSON 文件         | P1     |
| 导入JSON                      | 从 JSON 文件加载模型                                         | P1     |
| 导出CSV                       | 将当前视图的数据（包括节点名称、值、变化%）导出为 CSV 格式，供 Excel 使用 | P1     |
| **Excel模型迁移辅助**（亮点） | 提供模板或引导，帮助用户将简单的 Excel 模型结构（如单元格引用关系）映射到本工具的节点和公式 | P2     |

### 2.4 可视化与展示（“Stunning presentations”）

#### 2.4.1 节点可视化

| 功能     | 描述                                                         | 优先级 |
| :------- | :----------------------------------------------------------- | :----- |
| 节点卡片 | 每个节点以卡片形式展示：名称、当前值、单位、变化百分比、滑块（驱动因子） | P0     |
| 颜色编码 | 不同类型的节点用不同背景色区分（如驱动因子用浅蓝色，计算指标用白色） | P1     |
| 迷你图   | 如果有时间序列数据，在节点旁显示迷你趋势线（Sparkline）      | P2     |

#### 2.4.2 演示模式

| 功能     | 描述                                           | 优先级 |
| :------- | :--------------------------------------------- | :----- |
| 全屏模式 | 一键进入演示模式，隐藏编辑工具栏，仅展示画布   | P2     |
| 焦点模式 | 双击某节点，高亮显示其上下游路径，其余节点淡化 | P2     |
| 截图导出 | 将当前画布导出为图片（PNG），用于 PPT 汇报     | P2     |

### 2.5 高级规划功能（对标 ValQ 的深度场景）

#### 2.5.1 时间维度与滚动预测

| 功能         | 描述                                                         | 优先级 |
| :----------- | :----------------------------------------------------------- | :----- |
| 时间序列展开 | 节点数值可展开为时间维度（如月度、季度），每个时间点可独立编辑或通过公式关联 | P2     |
| 滚动预测     | 基于历史数据或线性趋势，自动生成未来 N 期的预测值            | P2     |
| 季节性调整   | 支持设置季节性系数，应用于预测                               | P3     |

#### 2.5.2 分摊与汇总

| 功能           | 描述                                               | 优先级 |
| :------------- | :------------------------------------------------- | :----- |
| 自上而下分摊   | 将顶层目标按权重分摊到底层驱动因子                 | P2     |
| 自下而上汇总   | 底层驱动因子变化自动汇总到顶层（已通过依赖图实现） | P1     |
| 自定义分摊权重 | 允许用户手动设置分摊权重                           | P2     |

### 2.6 数据协同与安全（Web版可扩展的方向）

| 数据输入与验证 | 允许多个利益相关方输入或调整自己负责的驱动因子，并提交“待验证”版本 | P3             |

## 4. 开发路线图（建议迭代顺序）

### 阶段一：MVP 核心（1-2周）——实现 P0 功能

1. **基础框架**：项目初始化，画布区域，节点添加/删除
2. **节点类型**：驱动因子 + 简单计算节点（加减乘除）
3. **依赖关系**：通过公式引用建立依赖，实现响应式更新
4. **UI交互**：滑块调整，实时计算
5. **视觉反馈**：绿/红颜色标识变化
6. **数据持久化**：导出/导入 JSON（备份模型）

**可演示场景**：简单的利润模型（收入-成本）或销售漏斗模型

### 阶段二：情景管理与 Excel 对接（2-3周）——实现 P1 功能

1. **情景功能**：保存、加载、设置基准
2. **公式增强**：支持括号、SUM、AVG 等函数
3. **画布增强**：缩放平移
4. **CSV 导出**：供 Excel 进一步分析
5. **环形依赖检测**

**可演示场景**：多情景对比（乐观 vs 保守），与 Excel 交互

### 阶段三：高级可视化与演示（2-3周）——实现 P2 功能

1. **视觉优化**：节点卡片美化，颜色编码
2. **演示模式**：全屏、焦点高亮
3. **截图导出**
4. **分摊功能**（自上而下）

**可演示场景**：完整销售规划、人力规划，可直接用于汇报

### 阶段四：时间维度与扩展（4-6周，可选）

1. **时间序列展开**
2. **滚动预测**
3. **协同与回写原型**（如需后端）

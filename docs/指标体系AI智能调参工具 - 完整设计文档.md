## 二、关键决策逻辑（完整版）

### 核心原则：取命中与兜底的并集

不单纯依赖知识库或兜底策略，而是取两者的最优组合，综合排序后输出最终方案。

### 决策逻辑详解

#### 1. 知识库匹配（命中集H）

- 从知识库中检索相似场景
- 得到命中结果集 H = {h₁, h₂, ..., hₘ}
- 每个命中结果包含：因子名称、调整方案、历史效果、相似度分数

#### 2. 兜底策略计算（候选集F）

当0命中或需要补充时执行：

- 敏感性高的正相关Top5
- 负相关Top5
- 目标标准差和波动标准差在改进区的优先调整
- 当前值与目标值差距大的驱动因子优先调整
- 以一级指标达标为目标

#### 3. 取并集 U = H ∪ F

合并命中集和候选集，去重后得到综合候选集

#### 4. 冲突处理与融合规则

| 场景             | 处理规则                                                     | 示例                             |
| :--------------- | :----------------------------------------------------------- | :------------------------------- |
| 因子只在H中      | 直接采用知识库的调整方案                                     | H有出价系数调至1.5 → 采用        |
| 因子只在F中      | 采用兜底策略的调整方案                                       | F有预算调至6000 → 采用           |
| 因子同时在H和F中 | 1. 优先采用知识库的调整方向 2. 调整幅度取加权平均（知识库权重0.7，兜底权重0.3） 3. 理由合并说明 | H建议+0.3，F建议+0.2 → 最终+0.27 |

#### 5. 综合优先级重排

计算每个因子的最终优先级分数：

| 来源     | 优先级计算公式                                     | 说明                          |
| :------- | :------------------------------------------------- | :---------------------------- |
| 仅知识库 | P = 0.8 + (相似度 × 0.2)                           | 基础分0.8，相似度越高加分越多 |
| 仅兜底   | P = (敏感度×0.4) + (改进区?1:0×0.3) + (差距分×0.3) | 三项加权                      |
| 同时出现 | P = max(知识库分, 兜底分) × 1.1                    | 给予10%融合加成               |

按P值从高到低排序，取前N个作为最终方案（N≤3，避免一次调整过多因子）

## 三、兜底策略的详细执行流程

### 步骤1：数据准备

- 获取所有驱动因子列表
- 获取敏感性分析数据（敏感度分数、相关性方向）
- 获取标准差分析数据（改进区/危险区标记）
- 获取目标差距分析数据（差距大小、是否显著）

### 步骤2：因子初筛

- 从所有驱动因子中，筛选出敏感性排名前20的因子
- 剔除处于危险区的因子

### 步骤3：计算目标差距

#### 3.1 驱动因子有直接目标的情况

text

复制下载

```
绝对差距 = 目标值 - 当前值
相对差距 = |绝对差距| / max(|目标值|, 1)
差距是否显著 = 相对差距 > 0.1 (10%)
方向是否对齐 = (需要提升且绝对差距>0) 或 (需要降低且绝对差距<0)
```



#### 3.2 驱动因子无直接目标的情况

根据一级指标差距和敏感性反推理想位置：

text

复制下载

```
如果 需要提升 且 正相关:
    理想位置 = 当前值 + (范围上限 - 当前值) × 敏感度
如果 需要提升 且 负相关:
    理想位置 = 当前值 - (当前值 - 范围下限) × 敏感度
如果 需要降低 且 正相关:
    理想位置 = 当前值 - (当前值 - 范围下限) × 敏感度
如果 需要降低 且 负相关:
    理想位置 = 当前值 + (范围上限 - 当前值) × 敏感度

相对差距 = |理想位置 - 当前值| / (范围上限 - 范围下限)
```



### 步骤4：多维度综合排序

计算每个候选因子的**综合优先级分数**，权重分配：

| 考量维度     | 权重 | 说明                       |
| :----------- | :--- | :------------------------- |
| 敏感度分数   | 40%  | 敏感性越高，调整效果越明显 |
| 是否在改进区 | 30%  | 改进区的因子调整风险小     |
| 目标差距大小 | 30%  | 差距越大，优化空间越大     |

**综合优先级分数 = 敏感度×0.4 + (改进区?1:0)×0.3 + 相对差距×0.3**

### 步骤5：分方向筛选

- **正相关因子**：按综合优先级分数排序，取Top5
- **负相关因子**：按综合优先级分数排序，取Top5

## 四、并集决策的详细执行流程

### 步骤1：接收输入

- 接收知识库命中的因子列表 H = [因子A, 因子B, ...]（带调整方案）
- 接收兜底策略筛选的因子列表 F = [因子C, 因子D, ...]（带调整建议）

### 步骤2：合并去重

text

复制下载

```
U = list(set(H + F))  # 取并集
```



### 步骤3：冲突处理

#### 3.1 为每个因子标记来源

- 来源类型：'H_only'、'F_only'、'both'
- 记录各来源的调整建议

#### 3.2 融合规则表

| 来源类型 | 处理规则       | 调整幅度计算                        | 理由生成模板                                                 |
| :------- | :------------- | :---------------------------------- | :----------------------------------------------------------- |
| H_only   | 直接采纳知识库 | 新值 = 知识库建议值                 | "参考历史经验：{历史效果}，建议{调整}"                       |
| F_only   | 直接采纳兜底   | 新值 = 兜底建议值                   | "基于敏感性分析{敏感度}和目标差距{差距}，建议{调整}"         |
| both     | 加权融合       | 新值 = 原值 + (Δ_kb×0.7 + Δ_fb×0.3) | "结合历史经验（{效果}）和当前分析（差距{差距}），建议{调整}" |

### 步骤4：综合优先级重排

计算每个因子的最终优先级分数：

#### 4.1 仅知识库来源

text

复制下载

```
P_h = 0.8 + (相似度 × 0.2)
```



#### 4.2 仅兜底来源

text

复制下载

```
P_f = (敏感度 × 0.4) + (改进区 ? 1 : 0 × 0.3) + (相对差距 × 0.3)
```



#### 4.3 同时出现

text

复制下载

```
P_both = max(P_h, P_f) × 1.1  # 融合加成10%
```



### 步骤5：最终筛选

- 按P值降序排序
- 取Top N（N≤3，可配置）
- 确保不超过系统设定的最大调整因子数

## 七、决策示例（完整场景）

### 场景设定

- **一级指标**：GMV，当前100万，目标120万（差距20%，需要提升）
- **用户需求**："帮我提升GMV，但要控制成本，不要太激进"

### 驱动因子数据

| 因子     | 当前值 | 目标值 | 范围             | 敏感度  | 相关性 | 改进区 | 差距 |
| :------- | :----- | :----- | :--------------- | :------ | :----- | :----- | :--- |
| 出价系数 | 1.2    | 1.5    | [0.5,2.0]        | 0.9(高) | 正相关 | ✅      | +25% |
| 预算上限 | 5000   | 6000   | [1000,10000]     | 0.5(中) | 正相关 | ✅      | +20% |
| 定向精度 | 广泛   | 核心   | [广泛,核心,精准] | 0.3(低) | 正相关 | ❌      | -    |
| 频次控制 | 3次    | 2次    | [1,5]            | 0.7(高) | 负相关 | ✅      | -33% |

### 知识库命中（H）

找到2个相似案例：

- **案例1**：相似度0.85，调整出价系数1.2→1.5，GMV提升15%，转化率微降
- **案例2**：相似度0.7，调整定向精度广泛→核心，GMV提升8%，成本略增

命中集 H = {出价系数(1.2→1.5), 定向精度(广泛→核心)}

### 兜底策略计算（F）

**综合优先级计算：**

| 因子     | 敏感度(0.4)  | 改进区(0.3) | 差距(0.3)      | 综合分    |
| :------- | :----------- | :---------- | :------------- | :-------- |
| 出价系数 | 0.9×0.4=0.36 | 1×0.3=0.3   | 0.25×0.3=0.075 | **0.735** |
| 频次控制 | 0.7×0.4=0.28 | 1×0.3=0.3   | 0.33×0.3=0.099 | **0.679** |
| 预算上限 | 0.5×0.4=0.20 | 1×0.3=0.3   | 0.20×0.3=0.06  | **0.560** |
| 定向精度 | 0.3×0.4=0.12 | 0×0.3=0     | 0×0.3=0        | **0.120** |

**正相关Top5**：出价系数(0.735)、预算上限(0.560)
**负相关Top5**：频次控制(0.679)

候选集 F = {出价系数, 频次控制, 预算上限}

### 并集处理

**步骤1：取并集**
U = {出价系数, 定向精度, 频次控制, 预算上限}

**步骤2：冲突处理**

| 因子     | 来源   | 知识库建议    | 兜底建议         | 融合方案        |
| :------- | :----- | :------------ | :--------------- | :-------------- |
| 出价系数 | both   | 1.2→1.5(+0.3) | 1.2→1.4(+0.2)    | 1.2→1.47(+0.27) |
| 频次控制 | F_only | -             | 3→2.5(-0.5)      | 3→2.5(-0.5)     |
| 预算上限 | F_only | -             | 5000→6000(+1000) | 5000→6000       |
| 定向精度 | H_only | 广泛→核心     | -                | 广泛→核心       |

**步骤3：优先级重排**

| 因子     | 来源   | 知识库分          | 兜底分 | 最终分                        |
| :------- | :----- | :---------------- | :----- | :---------------------------- |
| 出价系数 | both   | 0.8+0.85×0.2=0.97 | 0.735  | max(0.97,0.735)×1.1=**1.067** |
| 频次控制 | F_only | -                 | 0.679  | **0.679**                     |
| 预算上限 | F_only | -                 | 0.560  | **0.560**                     |
| 定向精度 | H_only | 0.8+0.7×0.2=0.94  | -      | **0.94**                      |

排序：出价系数(1.067) > 定向精度(0.94) > 频次控制(0.679) > 预算上限(0.560)

**步骤4：最终方案**（取Top 3，用户要求"不要太激进"所以选3个保守调整）

| 因子     | 调整前 | 调整后 | 来源   | 理由                                              |
| :------- | :----- | :----- | :----- | :------------------------------------------------ |
| 出价系数 | 1.2    | 1.47   | 融合   | 历史经验(相似场景提效15%) + 当前差距25%，加权融合 |
| 定向精度 | 广泛   | 核心   | 知识库 | 参考历史成功案例，相似度0.7                       |
| 频次控制 | 3次    | 2.5次  | 兜底   | 高敏感负相关+改进区+差距33%，温和下调             |

- 成本略有增加，但在可控范围内

## 九、FAQ

### Q1：为什么要取并集而不是只用命中或只用兜底？

**A**：知识库提供经验智慧，兜底策略提供数据分析，两者互补。并集策略能：

- 避免错过知识库中未覆盖但实际有效的因子
- 用兜底策略验证和修正知识库的建议
- 当知识库命中的因子不够时，用兜底策略补充

### Q2：同一个因子在命中和兜底中建议冲突怎么办？

**A**：采用加权融合：

- 如果方向一致（都建议上调/下调）：幅度加权平均
- 如果方向相反：以知识库为准（经验优先），但幅度保守（减半执行），并给出风险提示

### Q3：驱动因子的目标值从哪里来？

**A**：有两种来源：

1. 业务预设的目标值（如预算目标、出价上限等）
2. 根据一级指标目标反推的理论值（通过敏感性系数计算理想位置）

### Q4：差距大但敏感度低的因子要调整吗？

**A**：不建议。差距大但敏感度低，说明调整它效果不明显，属于"吃力不讨好"。应该优先选差距大且敏感度高的因子。

### Q5：一次最多调整几个因子？

**A**：建议不超过3个。调整因子太多会：

- 难以区分每个因子的实际效果
- 可能产生因子间的相互干扰
- 风险难以控制

### Q6：如何判断"改进区"？

**A**：基于标准差分析：

- 改进区：因子波动适中，有优化空间
- 危险区：波动过大，调整风险高
- 稳定区：波动过小，调整惰性强

### Q7：知识库如何构建和更新？

**A**：

- 初始阶段：导入历史调参记录
- 运行阶段：每次成功调参后自动存入
- 定期评估：剔除效果差的记录

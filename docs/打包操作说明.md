# AIDM 打包操作说明

## 环境要求

| 项目 | 要求 |
|------|------|
| Node.js | >= 18.x |
| npm | >= 9.x |
| 操作系统 | Windows 10/11 (x64) |

## 打包步骤

### 1. 安装依赖

```bash
cd "E:\MY AI\ValQ"
npm install
```

### 2. 构建前端

```bash
npm run build
```

成功后会在 `dist/` 目录下生成 `index.html`、`assets/` 等文件。

### 3. 执行打包（Windows 安装包）

```bash
npx electron-builder --win
```

打包成功后，`release/` 目录下会生成：

| 文件 | 大小 | 说明 |
|------|------|------|
| `AIDM Setup 1.0.0.exe` | ~141MB | NSIS 安装包 |
| `latest.yml` | ~150B | 自动更新配置文件 |
| `*.blockmap` | ~150KB | 增量更新映射文件 |
| `win-unpacked/` | ~500MB | 解压后的完整应用目录（可独立运行，无需安装） |

### 4. 验证安装包

打包完成后，可以双击 `release/win-unpacked/AIDM.exe` 直接测试（无需安装），确认功能正常后再分发安装包。

## 一键打包（完整流程）

```bash
cd "E:\MY AI\ValQ"
npm install && npm run build && npx electron-builder --win
```

## 打包产物说明

- **安装包** (`AIDM Setup 1.0.0.exe`): 用于分发给最终用户，内含 NSIS 安装向导
- **win-unpacked/**: 解压后的完整应用，可复制到任何位置直接运行 `AIDM.exe`，适合开发测试或便携使用
- **PBI Proxy**: `pbi-proxy.exe` 已自动打包进安装包，用户安装后自动释放到 `resources/pbi-proxy/` 目录

## 安全加固说明

打包后（生产模式）会自动启用以下安全机制：

| 机制 | 说明 |
|------|------|
| 禁用 DevTools | F12、Ctrl+Shift+I 快捷键被禁用 |
| 禁用右键菜单 | 右键上下文菜单被禁用 |
| 隐藏菜单栏 | 不显示 File/Edit/View 等默认菜单 |
| 授权验证 | AES-256-GCM 加密 + RSA-2048 签名校验 |
| 试用版防护 | SHA-256 哈希完整性校验，防止篡改 |

## 常见问题

### Q: 打包时卡在下载 Electron 运行时

**A**: 本地已有 Electron 缓存（`node_modules/electron/dist/`），如果仍然尝试从 GitHub 下载，可能是网络问题。可以尝试设置环境变量：

```bash
set ELECTRON_SKIP_BINARY_DOWNLOAD=1
npx electron-builder --win
```

### Q: 打包时报 rcedit 错误

**A**: `rcedit` 用于设置 exe 文件的版本信息，偶发失败但不影响功能。可以重新执行打包命令，通常会成功。

### Q: 如何打包其他平台？

**A**: 当前仅配置了 Windows 平台。如需打包 macOS 或 Linux，需要修改 `electron-builder.yml` 中的 `win` 配置为对应平台配置。

### Q: 如何更新版本号？

**A**: 修改 `package.json` 中的 `version` 字段，然后重新打包。安装包名称会自动带上版本号。

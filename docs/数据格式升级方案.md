# AIDM 数据格式升级方案

**版本**: 2.0
**更新日期**: 2026-03-25
**状态**: 方案确认中

---

## 一、核心设计决策

| 决策点 | 最终方案 |
|--------|---------|
| **长表格式** | `指标 ID，指标名称，AC, FC, BU, 时间` |
| **横表格式** | `指标 ID，指标名称，属性，202601, 202602, ...` |
| **时间维度** | 必须统一（年/季/月/周/日，不允许混合） |
| **滚动周期** | ✅ 允许跨年滚动（18 个月、24 个月等） |
| **三表合并** | FULL OUTER JOIN（缺失值填 NULL） |
| **三表结构** | 必须一致（指标 ID, 指标名称，时间，值） |
| **公式表关联** | 自动关联（通过指标 ID） |
| **属性列名** | 固定为 AC/FC/BU（校验不通过则报错） |
| **AC/FC/BU 分布** | AC 和 FC 互斥，BU 全期间存在 |
| **函数设计** | `PERIOD_SUM(表达式，属性，周期)` 替代 `MONTHLY_*` |
| **旧格式兼容** | ❌ 不兼容，一步到位升级 |
| **数据结构** | `originalTimeData` key 格式：`{time}-{attribute}` |

## 二、数据格式详解

### 2.1 横表格式（用户导入/导出）

```csv
指标 ID，指标名称，属性，202601,202602,202603,202604,202605,202606
yingyeshouru，营业收入，AC,1250000,1380000,1420000,1560000,1480000,1620000
yingyeshouru，营业收入，FC,1650000,1750000,1820000,1900000,1850000,1950000
yingyeshouru，营业收入，BU,1800000,1900000,2000000,2100000,2200000,2300000
yingyechengben，营业成本，AC,780000,850000,880000,960000,910000,1000000
```

**特点**：
- 每个指标 **3 行**（AC/FC/BU 各一行）
- 时间列为字符串（支持跨年滚动）
- 属性列固定为 `AC` | `FC` | `BU`

---

### 2.2 长表格式（系统存储 + Power BI 连接）

```csv
指标 ID，指标名称，AC, FC, BU, 时间
yingyeshouru，营业收入，1250000, 1650000, 1800000, 202601
yingyeshouru，营业收入，1380000, 1750000, 1900000, 202602
yingyechengben，营业成本，780000, NULL, 990000, 202601
```

**特点**：
- AC/FC/BU 作为列名
- 每行 = 一个指标在某个时间点的全部属性值
- 允许 NULL 值（FULL OUTER JOIN 结果）

---

### 2.3 三表模式合并逻辑（Power BI 连接）

当 AC/FC/BU 是三张独立表时，使用 FULL OUTER JOIN 合并：

```sql
SELECT
  COALESCE(ac.指标 ID, fc.指标 ID, bu.指标 ID) AS 指标 ID,
  COALESCE(ac.指标名称，fc.指标名称，bu.指标名称) AS 指标名称，
  ac.值 AS AC,
  fc.值 AS FC,
  bu.值 AS BU,
  COALESCE(ac.时间，fc.时间，bu.时间) AS 时间
FROM AC 表 ac
FULL OUTER JOIN FC 表 fc
  ON ac.指标 ID = fc.指标 ID AND ac.时间 = fc.时间
FULL OUTER JOIN BU 表 bu
  ON ac.指标 ID = bu.指标 ID AND ac.时间 = bu.时间
ORDER BY 指标 ID, 时间
```

**典型场景**：
| 场景 | AC | FC | BU | 处理方式 |
|------|-----|-----|-----|---------|
| 历史数据 | ✅ 有值 | ❌ NULL | ✅ 有值 | BU 作为 baseline |
| 预测数据 | ❌ NULL | ✅ 有值 | ✅ 有值 | AC 为空正常 |
| 完整数据 | ✅ 有值 | ✅ 有值 | ✅ 有值 | 理想状态 |

## 三、时间格式规范

### 3.1 格式定义表

| 类型 | 格式 | 示例 | 正则 | 长度 |
|------|------|------|------|------|
| 年度 | YYYY | `2026` | `^\d{4}$` | 4 |
| 季度 | YYYYQ# | `2026Q1` | `^\d{4}Q[1-4]$` | 6 |
| 月度 | YYYYMM | `202601` | `^\d{6}$` | 6 |
| 周度 | YYYYWK## | `2026WK01` | `^\d{4}WK\d{2}$` | 8 |
| 日报 | YYYYMMDD | `20260115` | `^\d{8}$` | 8 |

---

### 3.2 滚动周期示例

| 类型 | 正常周期 | 滚动周期（跨年） |
|------|---------|-----------------|
| **日报** | 20260101-20261231 | 20261201-20270115（46 天） |
| **周度** | 2026WK01-2026WK52 | 2026WK40-2027WK10（23 周） |
| **月度** | 202601-202612 | 202601-202706（18 个月） |
| **季度** | 2026Q1-2026Q4 | 2026Q2-2027Q3（8 季） |
| **年度** | 2026 | 2024, 2025, 2026（3 年） |

---

### 3.3 校验规则

- ✅ 允许跨年度：`202601, 202602, ..., 202612, 202701, 202702`（18 个月滚动）
- ❌ 不允许混维度：`2026, 2026Q1, 202601`（年 + 季 + 月 混合）

## 四、AC/FC/BU 分布规则

### 4.1 数据分布逻辑

```
时间轴：1 月 --- 8 月 --- 9 月 --- 12 月
        │实际期间│预测期间│

AC（实际）：只在已发生的期间存在（1-8 月）
FC（预测）：只在未发生的期间存在（9-12 月）
BU（目标）：全期间都有（1-12 月）
```

### 4.2 示例数据（月度 12 期）

```javascript

  '202601-AC': 1250000,  '202601-BU': 1800000,
  '202602-AC': 1380000,  '202602-BU': 1900000,
  '202603-AC': 1420000,  '202603-BU': 2000000,
  '202604-AC': 1560000,  '202604-BU': 2100000,
  '202605-AC': 1480000,  '202605-BU': 2200000,
  '202606-AC': 1620000,  '202606-BU': 2300000,
  '202607-AC': 1590000,  '202607-BU': 2400000,
  '202608-AC': 1710000,  '202608-BU': 2500000,

  // 9-12 月：只有 FC + BU（预测未发生，无实际）
  '202609-FC': 1650000,  '202609-BU': 2050000,
  '202610-FC': 1750000,  '202610-BU': 2150000,
  '202611-FC': 1820000,  '202611-BU': 2250000,
  '202612-FC': 1900000,  '202612-BU': 2350000,

```

### 4.3 计算逻辑

```javascript
// 当前值 = AC 汇总 + FC 汇总
const actualTotal = 1250000 + 1380000 + ... + 1710000;  // 1-8 月实际
const forecastTotal = 1650000 + 1750000 + ... + 1900000;  // 9-12 月预测
value = actualTotal + forecastTotal;  // = 19,630,000

// 基准值 = BU 汇总（全期间）
baseline = 1800000 + 1900000 + ... + 2350000;  // = 25,800,000

// 完成率 = (AC+FC) / BU
completionRate = value / baseline;  // = 76.1%
```

## 六、PERIOD 函数设计（替代 MONTHLY）

### 6.1 函数语法

```javascript
// 函数签名
PERIOD_SUM(表达式，属性，周期类型)
PERIOD_AVG(表达式，属性，周期类型)
PERIOD_MIN(表达式，属性，周期类型)
PERIOD_MAX(表达式，属性，周期类型)
PERIOD_COUNT(表达式，属性，周期类型)

// 参数说明：
// - 表达式：必填，计算公式或节点 ID
// - 属性：可选，'AC'|'FC'|'BU'|'AC+FC'，默认为空（自动汇总 AC+FC）
// - 周期类型：可选，'D'|'W'|'M'|'Q'|'Y'（日/周/月/季/年），默认为空（自动识别）
```

### 6.2 使用示例

```javascript
// 默认用法（推荐）
PERIOD_SUM(营业收入)         // 汇总 AC+FC（周期自动识别）

// 指定属性筛选
PERIOD_SUM(营业收入，'AC')    // 只汇总实际
PERIOD_SUM(营业收入，'FC')    // 只汇总预测
PERIOD_SUM(营业收入，'BU')    // 只汇总目标

// 指定周期类型
PERIOD_SUM(营业收入，, 'M')   // 月维度汇总
PERIOD_SUM(营业收入，, 'W')   // 周维度汇总
PERIOD_SUM(营业收入，, 'D')   // 日维度汇总

// 组合使用
PERIOD_SUM(营业收入，'AC', 'M')  // 月维度，只汇总实际
```

### 6.3 旧语法处理

| 旧语法 | 新语法 | 状态 |
|--------|--------|------|
| `MONTHLY_SUM(表达式)` | `PERIOD_SUM(表达式)` | ❌ 删除 |
| `MONTHLY_AVG(表达式)` | `PERIOD_AVG(表达式)` | ❌ 删除 |

**实施后 `MONTHLY_*` 函数完全移除，不再保留。**

---

### 6.4 计算逻辑详解

#### 当前 MONTHLY 函数逻辑（已验证）

```javascript
// calculateMonthlyValue() 核心流程：

// 步骤 1：对每个周期单独计算内部表达式
periodKeys.forEach(periodKey => {
  // 单周期值 = 内部表达式计算结果
  periodValue = innerCompileFn(monthValues);

  // 如果有外层公式（如 "/100"），也应用到周期值
  if (hasOuterFormula) {
    periodValue = outerCompileFn({ [placeholder]: periodValue });
  }

  timeData[periodKey] = periodValue;  // 存储周期值
  periodValuesArray.push(periodValue);
});

// 步骤 2：按聚合类型汇总
innerAggregatedValue = aggregateMonthlyValues(periodValuesArray, aggregationType);

// 步骤 3：如果有外层公式，对聚合后的值再应用一次
finalValue = outerCompileFn({ [placeholder]: innerAggregatedValue });
```

#### 新方案 PERIOD 函数逻辑

```javascript
// calculatePeriodValue() 核心流程：

// 步骤 1：属性筛选（新增）
let filteredKeys = [];
if (属性 === 'AC+FC' || 属性 === '') {
  // 默认：筛选 AC 和 FC（排除 BU）
  filteredKeys = allKeys.filter(k => k.endsWith('-AC') || k.endsWith('-FC'));
} else {
  // 指定属性
  filteredKeys = allKeys.filter(k => k.endsWith(`-${属性}`));
}

// 步骤 2：周期类型筛选（新增）
if (周期类型) {
  filteredKeys = filterByPeriodType(filteredKeys, 周期类型);
  // 例如：'M' → 只保留 6 位数字 (YYYYMM)
}

// 步骤 3-5：同当前逻辑
```

**关键差异**：
1. 新增属性筛选逻辑
2. 新增周期类型筛选逻辑
3. 核心计算逻辑保持一致（已验证可行）

## 七、普通表达式 vs PERIOD 函数

### 7.1 普通表达式（自动按月计算后聚合）

```javascript
// 毛利率 = 毛利润 / 营业收入
// 毛利润 = 营业收入 - 营业成本

// 计算逻辑：
// 1. 对每个月：毛利率 [月] = 毛利润 [月] / 营业收入 [月]
// 2. 按 aggregationType 聚合（默认 average）

// 适用场景：比率型指标（毛利率、净利率、费用率等）
```

### 7.2 PERIOD 函数（先计算表达式，再汇总）

```javascript
// 全年总毛利 = PERIOD_SUM(营业收入 - 营业成本)

// 计算逻辑：
// 1. 对每个月：毛利 [月] = 营业收入 [月] - 营业成本 [月]
// 2. 汇总 12 个月的毛利

// 适用场景：需要汇总的场景（总收入、总成本、总费用等）
```

### 7.3 混合使用

```javascript
// 年利润率 = 全年总利润 / 全年总收入
年利润率 = PERIOD_SUM(利润总额) / PERIOD_SUM(营业收入)

// 平均利润率 = 平均利润总额 / 平均营业收入
平均利润率 = 利润总额 / 营业收入  // 公式，aggregationType='average'
```

## 十三、普通表达式与函数计算逻辑

### 13.1 公式分类

| 类型 | 示例 | 计算时机 | 计算方式 |
|------|------|---------|---------|
| **普通表达式** | `A + B` | `_recalculate` 步骤 4 | `calculator.computeAll()` |
| **普通函数** | `SUM(A,B)`, `IF(A>0,B,0)` | 同上 | FormulaParser 编译为 JS 函数 |
| **周期函数** | `PERIOD_SUM(A*B)` | `_recalculate` 步骤 6 | `calculatePeriodValue()` 单独处理 |
| **混合表达式** | `PERIOD_SUM(A)/B` | 步骤 4 + 步骤 7 | placeholder 方式处理 |

### 13.2 普通表达式计算流程

```javascript
// 公式：毛利率 = 毛利润 / 营业收入

// 步骤 1：计算所有驱动因子的 value
nodeValues['毛利润'] = 5000000;
nodeValues['营业收入'] = 20000000;

// 步骤 2：编译公式
const compileFn = FormulaParser.compile('毛利润/营业收入', allNodeIds);

// 步骤 3：计算 value
nodeValues['毛利率'] = compileFn(nodeValues);  // = 0.25

// 步骤 4：计算 timeData（对每个月）
monthKeys.forEach(monthKey => {
  const monthValues = {
    '毛利润': nodeA.timeData[monthKey],
    '营业收入': nodeB.timeData[monthKey]
  };
  timeData[monthKey] = compileFn(monthValues);  // 月度毛利率
});
```

### 13.3 周期函数计算流程

```javascript
// 公式：全年总收入 = PERIOD_SUM(营业收入)

// 步骤 1：筛选 AC+FC（排除 BU）
const periodKeys = Object.keys(node.timeData)
  .filter(k => k.endsWith('-AC') || k.endsWith('-FC'));

// 步骤 2：对每个周期计算
periodKeys.forEach(periodKey => {
  periodValuesArray.push(node.timeData[periodKey]);
});

// 步骤 3：汇总
finalValue = sum(periodValuesArray);
```

### 13.4 混合表达式处理

```javascript
// 公式：月平均收入占比 = PERIOD_SUM(营业收入) / 12

// 步骤 1：用 placeholder 替换 PERIOD_SUM 部分
// '__PERIOD_SUM_RESULT__ / 12'

// 步骤 2：计算 PERIOD_SUM
const periodSum = calculatePeriodSum(营业收入);  // = 2000 万

// 步骤 3：应用外层公式
finalValue = periodSum / 12;  // = 166.67 万
```

### 13.2 标准差分析（StdDev Analysis）

**当前逻辑**：
```javascript
// 从 timeData 中提取每个月的值
months.forEach(month => {
  // 查找 key：'1 月实际'，'1 月预测'
  const value = timeData[findKey(month)];
  values.push(value);
});

// 计算标准差和变异系数
avg = sum(values) / values.length;
variance = sum(values.map(v => pow(v - avg, 2))) / values.length;
stdDev = sqrt(variance);
cvA = stdDev / avg;  // 变异系数 A（波动性）

// 计算目标偏离
deviations = values.map((v, i) => v - targets[i]);
cvB = stdDevB / targetAvg;  // 变异系数 B（目标达成度）

// 判定象限

```

**新方案修改**：

| 项目 | 当前 | 新方案 |
|------|------|--------|
| **key 查找** | `'1 月实际'` | `'202601-AC'` / `'202609-FC'` |
| **月份数组** | `['1 月', '2 月', ...]` | `['202601', '202602', ...]` |
| **周期类型** | 仅支持月度 | 支持 D/W/M/Q/Y |
| **数据模式** | mixed/actual-only/forecast-only | 不变 |

**修改后的代码**：
```javascript
function extractData(node, periods, dataMode) {
  const values = [];
  const targets = [];

  periods.forEach(period => {
    // 新格式：'202601-AC'
    const acKey = `${period}-AC`;
    const fcKey = `${period}-FC`;
    const buKey = `${period}-BU`;

    if (dataMode === 'mixed') {
      // 优先 AC，其次 FC
      if (node.originalTimeData[acKey] !== undefined) {
        values.push(node.originalTimeData[acKey]);
      } else if (node.originalTimeData[fcKey] !== undefined) {
        values.push(node.originalTimeData[fcKey]);
      }
    } else if (dataMode === 'actual-only') {
      if (node.originalTimeData[acKey] !== undefined) {
        values.push(node.originalTimeData[acKey]);
      }
    } else if (dataMode === 'forecast-only') {
      if (node.originalTimeData[fcKey] !== undefined) {
        values.push(node.originalTimeData[fcKey]);
      }
    }

    // 目标值
    if (node.originalTimeData[buKey] !== undefined) {
      targets.push(node.originalTimeData[buKey]);
    }
  });

  return { values, targets };
}
```

## 十六、AI 调参逆运算详解

### 16.1 逆运算场景分类

| 场景 | 用户输入 | 系统处理 |
|------|---------|---------|
| **分期调整** | "5 月收入增加 100 万" | 直接更新 `timeData['202605-AC']` |
| **总额调整** | "全年收入增加 1000 万" | 按权重/比例分配到各期 |
| **百分比调整** | "Q3 收入增长 10%" | 计算基数 × 1.1 |
| **目标导向** | "全年完成 1 亿收入" | 计算差额，按需分配 |

### 16.2 总额调整的逆运算逻辑

```javascript
/**
 * AI 调参逆运算引擎
 * @param {string} nodeId - 节点 ID
 * @param {number} totalAdjustment - 调整总额
 * @param {object} options - 配置选项
 */
function executeReverseCalculation(nodeId, totalAdjustment, options = {}) {
  const node = nodes[nodeId];
  const {
    distributionMode = 'proportional',  // 'proportional' | 'weight' | 'ai-smart'
    weights = {},                        // 权重配置
    periods = []                         // 调整期间列表
  } = options;

  // 获取当前分期数据
  const periodData = {};
  periods.forEach(period => {
    const acKey = `${period}-AC`;
    const fcKey = `${period}-FC`;
    if (node.timeData[acKey] !== undefined) {
      periodData[period] = node.timeData[acKey];
    } else if (node.timeData[fcKey] !== undefined) {
      periodData[period] = node.timeData[fcKey];
    }
  });

  // 计算当前总额
  const currentTotal = Object.values(periodData).reduce((a, b) => a + b, 0);

  // 根据分配模式计算各期调整额
  let adjustmentMap = {};

  if (distributionMode === 'proportional') {
    // 按比例分配（默认）
    Object.entries(periodData).forEach(([period, value]) => {
      const ratio = value / currentTotal;
      adjustmentMap[period] = totalAdjustment * ratio;
    });
  } else if (distributionMode === 'weight') {
    // 按权重分配
    const weightSum = Object.values(weights).reduce((a, b) => a + b, 0);
    if (Math.abs(weightSum - 1) > 0.0001) {
      throw new Error(`权重总和为${weightSum}，必须等于 1`);
    }
    Object.entries(weights).forEach(([period, weight]) => {
      adjustmentMap[period] = totalAdjustment * weight;
    });
  } else if (distributionMode === 'ai-smart') {
    // AI 智能分配（考虑季节性、趋势等）
    adjustmentMap = await AISmartDistribution(node, totalAdjustment, periods);
  }

  // 更新 timeData
  Object.entries(adjustmentMap).forEach(([period, adjustment]) => {
    const acKey = `${period}-AC`;
    const fcKey = `${period}-FC`;
    if (node.timeData[acKey] !== undefined) {
      node.timeData[acKey] += adjustment;
    } else if (node.timeData[fcKey] !== undefined) {
      node.timeData[fcKey] += adjustment;
    }
  });

  // 重新聚合 value
  recalculate();

  return { adjustmentMap, newTotal: currentTotal + totalAdjustment };
}
```

### 16.5 校验规则

```javascript
// AI 调参结果校验
function validateAIAdjustment(adjustments) {
  const errors = [];

  adjustments.forEach(adj => {
    // 1. 检查调整方向（正负）是否合理
    if (adj.adjustment > 0 && adj.reason?.includes('下降')) {
      errors.push(`调整方向与描述不符：${adj.nodeId}`);
    }

    // 2. 检查调整后是否超出合理范围
    const newValue = nodes[adj.nodeId].timeData[adj.key] + adj.adjustment;
    if (newValue < 0) {
      errors.push(`调整后值为负数：${adj.nodeId} ${adj.key}`);
    }

    // 3. 检查权重总和（如果有）
    if (adj.weights) {
      const weightSum = Object.values(adj.weights).reduce((a, b) => a + b, 0);
      if (Math.abs(weightSum - 1) > 0.0001) {
        errors.push(`权重总和为${weightSum}，必须等于 1`);
      }
    }
  });

  return { valid: errors.length === 0, errors };
}
```

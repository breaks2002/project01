# 测试数据说明

## 数据格式说明

所有测试数据文件均采用新横表格式：
- **列结构**：指标 ID，指标名称，属性，时间 1，时间 2，...
- **属性**：AC(实际)、FC(预测)、BU(目标)
- **时间格式**：
  - 月度：YYYYMM (如 202601)
  - 年度：YYYY (如 2023)
  - 季度：YYYYQ# (如 2024Q1)
  - 周度：YYYYWK## (如 2026WK01)
  - 日度：YYYYMMDD (如 20260101)

## 公式表文件

### 公式表结构
| 列名 | 说明 |
|------|------|
| 指标 ID | 指标唯一标识（英文） |
| 指标名称 | 中文名称 |
| 节点类型 | driver(驱动因子) 或 computed(计算指标) |
| 公式 | 计算公式（computed 类型必填） |
| 最小值 | 滑块最小值（driver 类型） |
| 最大值 | 滑块最大值（driver 类型） |
| 单位 | 元、% 等 |
| 显示格式 | 数字格式 |
| 指标方向 | auto(正向)、negative(反向) |
| 层级 | 1-4 级（1 级最高） |
| 汇总方式 | sum(加总)、average(平均) |
| 比率型指标 | TRUE/FALSE |

### 计算指标公式
- **毛利润** = 营业收入 - 营业成本
- **营业利润** = 毛利润 - 管理费用 - 销售费用
- **净利润** = 营业利润 × 0.75（假设税率 25%）

### 层级结构
```
1 级：净利润
2 级：营业利润
3 级：毛利润、管理费用、销售费用
4 级：营业收入、营业成本
```

## 导入测试流程

### 测试 1：基本导入（数据表 + 公式表）
1. 导入 `月度测试数据_12 期.csv`
2. 确认提示后，导入 `月度测试数据_公式表.csv`
3. 验证 7 个节点是否正确显示
4. 验证计算指标公式是否正确
5. 验证驱动因子的 min/max 是否合理

### 测试 2：月份调整
1. 点击任意驱动因子的"月份调整"
2. 修改预测月份的数值
3. 按回车或点击其他地方确认
4. 验证合计值是否更新
5. 验证节点卡片上的总值是否更新

### 测试 3：权重分配
1. 点击"权重分配"
2. 调整各月权重
3. 点击"应用分配"
4. 验证各月数值是否按权重重新分配

### 测试 4：趋势图
1. 点击节点的"📈"按钮
2. 验证是否正确显示所有月份的数据
3. 验证实际、预测、目标线是否正确

### 测试 5：不同时间维度
1. 分别导入年度、季度、周度、日度数据
2. 验证时间格式是否正确识别
3. 验证趋势图是否显示正确的时间轴

## 注意事项

1. **CSV 分隔符**：所有文件使用英文逗号分隔
2. **引号格式**：包含逗号的值（如 `"#,##0"`）使用双引号包裹
3. **空值处理**：实际期和预测期的空单元格表示该期间无数据
4. **公式表匹配**：公式表的指标 ID 必须与数据表一致
5. **min/max 设置**：公式表中的最小值/最大值应与实际数据相匹配

---

## 数据合理性检查

导入后可通过以下方式验证数据合理性：
1. 驱动因子的滑块范围应覆盖实际值和目标值
2. 计算指标的公式计算结果应与手动计算一致
3. 趋势图显示的数值应与节点卡片显示的汇总值一致

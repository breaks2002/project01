## 1. 概述

### 1.1 AI 决策模块简介

AI 决策模块是 VDT（驱动型业务规划工具）的核心功能之一，提供基于人工智能的业务指标智能调参能力。通过分析用户输入的业务背景和目标，AI 自动推导出需要调整的驱动因子及其调整幅度。

### 1.2 模块组成

AI 决策模块包含 6 个子模块：

| 模块 | 图标 | 功能描述 |
|------|------|----------|
| AI 配置 | ⚙️ | 配置 AI 连接参数、模型选择、温度值等 |
| AI 调参 | 🎯 | 智能调整驱动因子，生成调参方案 |
| 知识库 | 📚 | 管理历史调参案例，支持语义检索 |
| 场景选择 | 📋 | 选择 AI 场景模板，定义 Prompt 策略 |
| 规则管理 | 📋 | 配置约束映射规则和单位体系 |
| 别名管理 | 🏷️ | 配置因子别名映射，提升识别准确率 |

### 1.3 访问方式

1. 在顶部工具栏找到「AI 决策」按钮（紫色）
2. 点击后展开下拉菜单
3. 选择对应的模块进入

## 2. AI 配置

### 2.1 功能说明

配置 AI 服务连接参数，支持多家 AI 厂商和本地部署方案。

### 2.2 打开方式

**AI 决策** → **AI 配置**

### 2.3 配置项说明

#### 2.3.1 AI 调参配置（页签 1）

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 服务厂商 | 选择 AI 服务提供商 | 自定义 |
| API 地址 | AI 服务接口地址 | - |
| API 密钥 | 认证密钥 | - |
| 模型名称 | 使用的 AI 模型 | - |
| 温度值 | 生成随机性 (0-2) | 0.7 |
| 最大 Token 数 | 单次响应最大长度 | 2000 |
| System Prompt | 系统提示词模板 | - |

**温度值说明**：
- **0.0-0.3**：输出严谨、确定性强
- **0.4-0.7**：平衡创造性与准确性（推荐）
- **0.8-1.0**：更具创造性，适合头脑风暴
- **>1.0**：高度发散，可能不稳定

#### 2.3.2 AI 嵌入配置（页签 2）

用于知识库语义检索的向量嵌入模型配置。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用嵌入 | 是否启用向量嵌入 | 否 |
| API 地址 | 嵌入模型服务地址 | - |
| API 密钥 | 认证密钥 | - |
| 模型名称 | 嵌入模型名称 | - |
| 嵌入模型 | 具体嵌入模型 | text-embedding-ada-002 |
| 向量维度 | 输出向量维度 | 1536 |

### 2.4 预设厂商

系统内置以下厂商预设：

| 厂商 | 说明 |
|------|------|
| 自定义 | 手动配置所有参数 |
| DeepSeek | 深度求索 AI |
| 通义千问 | 阿里云 Qwen 模型 |
| 文心一言 | 百度 AI 模型 |
| 智谱 AI | 智谱华章 GLM 模型 |
| 讯飞星火 | 科大讯飞模型 |
| 本地 Ollama | 本地部署 Ollama |
| 本地 LM Studio | 本地部署 LM Studio |

### 2.5 操作步骤

#### 配置 AI 连接

1. 选择服务厂商（如选择「自定义」需手动填写）
2. 填写 API 地址（如：`https://api.deepseek.com/v1`）
3. 填写 API 密钥
4. 选择或填写模型名称（如：`deepseek-chat`）
5. 调整温度值和最大 Token 数（可选）
6. 点击「测试连接」验证配置
7. 测试通过后点击「保存配置」

#### 测试连接

1. 配置完成后点击「测试连接」按钮
2. 等待测试结果（约 2-5 秒）
3. 成功显示「✅ 连接成功！」
4. 失败显示错误信息，检查配置后重试

### 2.6 注意事项

- API 密钥会加密存储到本地 localStorage
- 修改配置后需重新测试连接
- 首次使用建议先用预设厂商配置，熟悉后再自定义
- 本地部署需确保服务已启动且地址正确

## 3. AI 调参

### 3.1 功能说明

AI 调参是核心功能，根据用户输入的业务背景和目标，智能分析并生成驱动因子调整方案。

### 3.2 打开方式

**AI 决策** → **AI 调参**

> ⚠️ **注意**：需先导入或创建指标模型才能使用此功能

### 3.3 界面布局

```
┌─────────────────────────────────────────────┐
│  AI 智能调参                            ✕   │
├─────────────────────────────────────────────┤
│  描述业务背景和目标                          │
│  ┌─────────────────────────────────────┐    │
│  │ 例如：Q4 是销售旺季，公司计划加大...   │    │
│  └─────────────────────────────────────┘    │
│                                             │
│  上传经营计划文档 📎                         │
│  已选场景 [财务场景]  已选知识库 [#制造业...] │
│  [选择场景] [选择知识库]                     │
│  ─────────────────────────────────────────  │
│  [知识库] [规则管理] [别名管理]              │
│                                             │
│  [✨ 开始智能分析]                          │
└─────────────────────────────────────────────┘
```

### 3.4 操作步骤

#### 步骤 1：描述业务背景

在文本框中输入业务场景和目标，例如：

```
Q4 是销售旺季，公司计划加大市场推广力度，销售费用可以适当增加用于广告投放。
同时我们希望优化管理费用，目标净利润增长 20%。
```

**输入提示**：
- 支持自然语言描述
- 可以包含多个目标
- 可以指定某些因子的调整方向
- 可以使用「控制在」「增加」「降低」等约束词

#### 步骤 2：因子匹配检测（实时）

输入时系统会自动检测：
- ✅ **已匹配模型因子**：显示识别到的因子名称
- ⚠️ **模型无此指标**：提示用户输入中存在的但模型中没有的指标

#### 步骤 3：上传经营计划文档（可选）

点击「上传经营计划文档」按钮，支持格式：
- PDF、Word、Excel、TXT、MD

系统会自动解析文档内容并提取关键信息。

#### 步骤 4：选择场景和知识库（可选）

- **选择场景**：点击按钮打开场景选择器，选择适用的场景模板
- **选择知识库**：点击按钮打开知识库，选择相关历史案例

#### 步骤 5：开始智能分析

点击「开始智能分析」按钮，等待 AI 分析：

1. **解析用户输入**：提取业务背景和约束条件
2. **规则引擎匹配**：匹配预定义的约束规则
3. **知识库检索**：检索相似历史案例（如果有）
4. **兜底策略分析**：基于敏感性、标准差等数据分析
5. **并集决策**：综合所有信息生成最终方案

#### 步骤 6：查看分析结果

分析完成后显示：

**AI 理解摘要**
- 业务背景总结
- 目标列表

**驱动因子分析表**
| 因子 | 当前值 | 目标值 | 差距 | 敏感度 | 相关性 | 改进区 | 优先级 |
|------|--------|--------|------|--------|--------|--------|--------|
| 销售费用 | 231 万 | - | - | 0.85 | 正相关 | 是 | 0.82 |

**知识库命中**（如果有）
- 显示相似历史案例及调整方案

**并集决策结果**
- Top 3 推荐调整因子
- 每个因子的调整幅度和理由

### 3.5 调整方案应用

1. 查看推荐方案
2. 勾选需要应用的因子（默认全选）
3. 点击「应用调整」
4. 系统会自动计算新的目标值并更新模型

### 3.6 特殊约束语法

系统支持以下约束表达：

| 类型 | 示例 | 说明 |
|------|------|------|
| 增加 | 毛利率增加 5 个百分点 | 指定增幅 |
| 降低 | 管理费用降低 10% | 指定降幅 |
| 控制 | 管理费用控制在 100 万以内 | 设置上限 |
| 允许 | 销售费用允许超出 20% | 允许突破限制 |
| 必须 | 净利润必须达到 500 万 | 强制目标 |
| 范围 | 销售费用在±10% 范围内调整 | 限定调整范围 |
| 稳定 | 尽量保持管理费用稳定 | 小幅波动 |

### 3.7 注意事项

- 确保已导入或创建指标模型
- 描述越详细，AI 理解越准确
- 可以多次调整输入，生成不同方案
- 约束条件用逗号分隔，避免跨句歧义

## 4. 知识库

### 4.1 功能说明

知识库用于存储和检索历史调参案例，支持多行业、多领域的经验积累和复用。

### 4.2 打开方式

**AI 决策** → **知识库**

### 4.3 界面布局

```
┌─────────────────────────────────────────────┐
│  📚 知识库                      [-] [✕]    │
├─────────────────────────────────────────────┤
│  [上传文档] [手动录入] [🔍 搜索...]         │
│  ─────────────────────────────────────────  │
│  ✅ 已选择：Q4 销售旺季推广方案               │
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │ ☐ 零售业 - 利润增长案例               │    │
│  │   销售费用→+30%, GMV→+18%           │    │
│  │   2026-03-14 | 相似度 0.85          │    │
│  ├─────────────────────────────────────┤    │
│  │ ☐ 制造业 - 成本优化案例               │    │
│  │   营业成本→-15%, 良率→+5%           │    │
│  │   2026-03-10 | 相似度 0.72          │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### 4.4 核心功能

#### 4.4.1 文档上传

**支持格式**：TXT、Word、PDF、Excel、MD

**操作步骤**：
1. 点击「上传文档」按钮
2. 选择文件（支持多选）
3. 等待解析完成（约 5-10 秒）
4. 填写元数据（行业、场景、标题等）
5. 确认因子调整方案和结果
6. 点击「保存到知识库」

**自动提取**：
- 场景关键词（如：销售旺季、利润增长）
- 因子调整记录（因子名称、调整幅度）
- 效果数据（结果指标变化）

#### 4.4.2 手动录入

适用于快速记录或修正自动提取结果。

**操作步骤**：
1. 点击「手动录入」按钮
2. 填写基本信息：
   - 标题：案例名称
   - 行业：所属行业
   - 场景：适用场景关键词
3. 添加因子调整记录：
   - 因子名称
   - 调整类型（增加/减少/控制）
   - 调整幅度
   - 结果说明
4. 点击「保存」

#### 4.4.3 语义检索

**搜索方式**：
1. 在搜索框输入关键词
2. 系统自动进行语义相似度检索
3. 返回最相关的 Top 3 案例

**检索原理**：
- 使用向量嵌入（Embedding）将文本转换为向量
- 计算查询向量与知识库向量的余弦相似度
- 返回相似度最高的案例

#### 4.4.4 案例管理

**编辑案例**：
1. 点击案例右侧的「编辑」按钮
2. 修改内容
3. 点击「保存更新」

**删除案例**：
1. 点击案例右侧的「删除」按钮
2. 确认删除

**批量操作**：
- 勾选多个案例
- 使用批量删除功能

### 4.5 数据结构

每个知识库条目包含：

```json
{
  "id": "kb_001",
  "title": "Q4 销售旺季推广方案",
  "industry": "零售/电商",
  "scenario": "销售旺季、利润增长",
  "factors": [
    {
      "factorName": "销售费用",
      "factorType": "expense",
      "adjustment": {
        "from": 80,
        "to": 105,
        "changePercent": 31.25
      },
      "result": "GMV 提升 18%, 净利润提升 15%"
    }
  ],
  "embedding": [0.123, -0.456, ...],
  "createdAt": "2026-03-14"
}
```

### 4.6 与 AI 调参的协同

**自动检索**：
- AI 调参时自动检索知识库
- 返回相似案例作为参考
- 融合历史经验到调参方案

**手动选择**：
1. 在 AI 调参界面点击「选择知识库」
2. 勾选相关案例
3. 系统会优先参考选中的案例

### 4.7 最佳实践

- **及时沉淀**：每次调参后保存有效方案
- **规范命名**：标题清晰，便于检索
- **分类管理**：按行业、场景分类
- **定期清理**：删除过时或无效的条目

## 5. 场景选择

### 5.1 功能说明

场景选择用于选择 AI 场景模板，定义 AI 分析的策略和 Prompt。

### 5.2 打开方式

**AI 决策** → **场景选择**

### 5.3 内置场景

系统内置以下场景模板：

| 场景 | 适用场景 | 说明 |
|------|----------|------|
| 财务场景 | 成本优化、利润提升、收入增长 | 财务指标调参 |
| HR 人力场景 | 招聘优化、人力成本控制、人效提升 | 人力资源调参 |
| 生产场景 | 产能提升、良率优化、生产计划 | 生产制造调参 |
| 通用场景 | 跨部门综合优化 | 默认场景 |

### 5.4 操作步骤

#### 选择场景

1. 点击「场景选择」按钮
2. 勾选适用的场景模板（支持多选）
3. 系统会自动应用到 AI 调参

#### 自定义场景

1. 点击「新建模板」按钮
2. 填写模板信息：
   - 名称：场景名称
   - 描述：适用场景说明
   - 关键词：用于自动匹配
   - System Prompt：提示词模板
3. 点击「保存」

#### 导入/导出模板

**导入**：
1. 点击「导入」按钮
2. 选择 MD 或 JSON 格式文件
3. 确认导入

**导出**：
1. 点击模板右侧的「导出」按钮
2. 保存 JSON 格式文件

### 5.5 Prompt 模板格式

**Markdown 格式示例**：

```markdown
# 财务场景

## 适用场景
- 成本优化
- 利润提升
- 收入增长

## System Prompt

你是一位财务分析专家，擅长通过驱动因子分析实现财务目标。

【核心任务】
1. 分析用户输入的财务目标
2. 识别关键驱动因子
3. 生成可行的调整方案

【强制要求】
- 优先考虑成本效益比
- 避免大幅波动
- 确保方案可执行
```

**JSON 格式示例**：

```json
{
  "name": "财务场景",
  "description": "成本优化、利润提升、收入增长",
  "keywords": ["成本", "利润", "收入", "财务"],
  "systemPrompt": "你是一位财务分析专家..."
}
```

### 5.6 自动匹配

系统会根据以下信息自动匹配场景：

- 用户输入的业务背景
- 模型中的因子名称
- 选择的知识点

### 5.7 注意事项

- 至少选择一个场景（推荐）
- 可以多选，系统会融合多个场景的策略
- 自定义场景会创建内置场景的副本

## 6. 规则管理

### 6.1 功能说明

规则管理用于配置约束映射规则，将用户自然语言转换为结构化约束条件。

### 6.2 打开方式

**AI 决策** → **规则管理**

或在 AI 调参窗口中点击「规则管理」按钮

### 6.3 界面布局

```
┌─────────────────────────────────────────────┐
│  📋 规则映射管理                      [✕]  │
├─────────────────────────────────────────────┤
│  [规则管理] [单位管理]                      │
├─────────────────────────────────────────────┤
│  [+] 添加规则  [🔍 搜索...]                 │
│  ─────────────────────────────────────────  │
│  ▼ 允许类                                   │
│  • 允许超出：允许/可/可以... → 超出/超过... │
│  • 控制限制：控制在/不超过/最多...          │
│  ─────────────────────────────────────────  │
│  ▼ 调整类                                   │
│  • 增加/增长：增加/增长/提升/提高...        │
│  • 降低/减少：降低/减少/削减/压缩...        │
│  ─────────────────────────────────────────  │
│  ▼ 强制类                                   │
│  • 必须达到：必须/需要/要/务必...           │
│  • 必须降低：必须/需要/要/务必...           │
└─────────────────────────────────────────────┘
```

### 6.4 规则配置

#### 添加规则

1. 点击「添加规则」按钮
2. 填写规则信息：
   - **规则名称**：如「允许超出」
   - **类别**：允许/调整/强制
   - **动作类型**：选择预定义类型
   - **关键词**：触发此动作的词（逗号分隔）
   - **触发词**：辅助触发的词（逗号分隔）
   - **启用**：是否启用此规则
3. 点击「保存」

#### 编辑规则

1. 点击规则右侧的「编辑」按钮
2. 修改配置
3. 点击「保存更新」

#### 删除规则

1. 点击规则右侧的「删除」按钮
2. 确认删除

### 6.5 单位管理

#### 添加单位

1. 切换到「单位管理」页签
2. 点击「添加单位」按钮
3. 填写单位信息：
   - **单位名称**：如「万元」
   - **单位类型**：absolute（绝对额）/ ratio（比率）
   - **关键词**：识别此单位的词（逗号分隔）
   - **换算倍数**：如「万」= 10000
   - **启用**：是否启用此单位
4. 点击「保存」

#### 内置单位

系统内置以下单位：

| 单位 | 类型 | 关键词 | 倍数 |
|------|------|--------|------|
| 万 | absolute | 万，万元 | 10000 |
| 亿 | absolute | 亿，亿元 | 100000000 |
| 元 | absolute | 元，块 | 1 |
| 百分比 | ratio | %，百分之 | 0.01 |
| 百分点 | ratio | 个百分点，个点 | 0.01 |

### 6.6 规则优先级

当多个规则匹配时，优先级顺序：

1. **强制类规则**（must_reach, must_reduce）
2. **控制类规则**（control_limit, allow_override）
3. **调整类规则**（increase, decrease）

### 6.7 与 AI 调参的协同

**规则引擎解析**：
1. 用户输入业务背景
2. 规则引擎匹配约束规则
3. 提取结构化约束条件
4. 传递给 AI 作为输入

**AI 兜底**：
- 规则引擎无法匹配时使用 AI 语义解析
- 确保复杂表达也能正确处理

### 7.5 后缀管理

#### 功能说明

后缀用于用户输入时的智能分词识别。例如输入「优化各项费用」，系统会提取包含「费用」后缀的词组。

#### 添加后缀

1. 切换到「后缀管理」页签
2. 点击「添加后缀」按钮
3. 填写后缀：
   - **后缀内容**：如「费用」
   - **启用**：是否启用此后缀
4. 点击「保存」

#### 内置后缀

系统内置以下后缀：

| 后缀 | 示例 |
|------|------|
| 费用 | 管理费用、销售费用 |
| 成本 | 营业成本、生产成本 |
| 收入 | 营业收入、其他收入 |
| 利润 | 毛利润、净利润 |
| 利率 | 毛利率、净利率 |
| 率 | 毛利率、良率 |
| 额 | 金额、总额 |

### 7.6 与 AI 调参的协同

**因子匹配检测**：
1. 用户输入业务背景
2. 系统使用别名表进行模糊匹配
3. 显示匹配结果和未匹配提示

**规则引擎解析**：
1. 规则引擎加载用户别名配置
2. 使用别名进行约束匹配
3. 提升识别准确率

### 7.7 最佳实践

- **优先使用标准名称**：减少歧义
- **合理配置别名**：确保是同义词
- **定期维护**：根据实际使用情况优化
- **避免过度泛化**：如「毛利率」≠「毛利润」

## 8. 常见问题

### 8.1 AI 连接问题

**Q: 测试连接失败，提示「网络错误」**

A: 检查以下几点：
1. API 地址是否正确（包含 `https://`）
2. API 密钥是否有效
3. 网络是否畅通
4. 防火墙是否阻止

**Q: 响应超时**

A: 尝试：
1. 增加超时时间设置（如有）
2. 选择更快的模型
3. 减少最大 Token 数

### 8.2 约束解析问题

**Q: 输入「毛利率增加 5 个百分点，管理费用控制在 100 万」，只解析到 1 条约束**

A: 可能原因：
1. 模型中没有「毛利率」这个因子（只有「毛利润」）
2. 解决方案：
   - 在模型中添加「毛利率」计算指标
   - 或在别名管理中将「毛利率」添加为「毛利润」的别名

**Q: 约束解析结果不正确**

A: 尝试：
1. 使用更明确的表达
2. 用逗号分隔不同约束
3. 在规则管理中添加自定义规则

### 8.3 知识库问题

**Q: 上传文档后解析失败**

A: 检查：
1. 文件格式是否支持
2. 文件是否损坏
3. 文件内容是否清晰（扫描件可能识别失败）
4. 尝试手动录入

**Q: 检索结果不相关**

A: 建议：
1. 使用更具体的关键词
2. 增加知识库条目数量
3. 配置嵌入模型提升语义检索效果

### 8.4 场景选择问题

**Q: 没有适合我的场景的模板**

A: 解决方案：
1. 使用「通用场景」
2. 自定义场景模板
3. 导入他人分享的模板

### 8.5 别名管理问题

**Q: 别名配置后没有生效**

A: 检查：
1. 是否启用了别名
2. 别名是否是真正的同义词
3. 刷新页面后重试

# AIDM 注册机操作说明书

##  概述

AIDM 注册机是用于生成和管理 AIDM 软件授权文件的工具集。包含两个主要工具：

| 工具 | 路径 | 用途 |
|------|------|------|
| 注册机 (license-gen) | `license-generator/license-gen.js` | 生成授权文件 (.lic) |
| 管理工具 (license-mgr) | `license-manager/license-mgr.js` | 管理授权记录、查询状态 |

---

## 🔑 首次使用 - 生成密钥对

**在使用注册机之前，需要先生成 RSA 密钥对。**

```bash
cd "E:\MY AI\ValQ\license-generator"
node license-gen.js --generate-keys
```

生成后会在 `keys` 目录下创建两个文件：
- `private.pem` - 私钥（️ **机密文件，仅供管理员使用，切勿泄露**）
- `public.pem` - 公钥（已嵌入 AIDM 客户端，用于验证授权文件签名）

> 📌 **重要提示**：生成密钥对后，请立即备份整个 `keys` 文件夹。如果私钥丢失，将无法再为已授权的用户生成新的授权文件。

## 🛠️ 注册机 (license-gen) 使用方法

### 基本用法

```bash
node license-gen.js --machine <机器码> --type <授权类型> --duration <有效期> [--output <输出路径>]
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `--machine` | 是 | 用户机器码（32位十六进制字符，如 `88594C5EF7B300AAE81A8337FE014C2F`） |
| `--type` | 是 | 授权类型：`trial` / `standard` / `pro` |
| `--duration` | 是 | 有效期格式：`30d`(天) / `1m`(月) / `1y`(年) / `10y`(十年) |
| `--output` | 否 | 输出文件路径，默认生成到当前目录 |

### 授权类型说明

| 类型 | 名称 | 功能 |
|------|------|------|
| `trial` | 试用版 | 基础功能，节点≤30，层级≤3，禁用AI/导出/PowerBI |
| `standard` | 标准版 | 基础+高级功能 |
| `pro` | 专业版 | 全部功能（AI决策、导出、PowerBI连接等） |

### 有效期格式

| 格式 | 含义 | 示例 |
|------|------|------|
| `Nd` | N 天 | `30d` = 30天 |
| `Nm` | N 月 | `3m` = 3个月 |
| `Ny` | N 年 | `1y` = 1年, `10y` = 10年 |

### 使用示例

#### 1. 生成 30 天专业版授权

```bash
cd "E:\MY AI\ValQ\license-generator"
node license-gen.js --machine 88594C5EF7B300AAE81A8337FE014C2F --type pro --duration 30d
```

#### 2. 生成 1 年标准版授权，指定输出路径

```bash
node license-gen.js --machine 88594C5EF7B300AAE81A8337FE014C2F --type standard --duration 1y --output C:\Users\shifei2\Desktop\license.lic
```

#### 3. 生成 10 年专业版授权

```bash
node license-gen.js --machine 88594C5EF7B300AAE81A8337FE014C2F --type pro --duration 10y
```

### 输出示例

```
✅ AIDM 授权文件已生成！
   机器码: 88594C5EF7B300AAE81A8337FE014C2F
   类型: pro
   有效期: 30d (到期: 2026/06/26)
   功能: basic, advanced, ai
   文件: 88594C5EF7B300AAE81A8337FE014C2F-pro-30d.lic
```

## 📊 管理工具 (license-mgr) 使用方法

管理工具有两种使用方式：**交互模式** 和 **命令行模式**。

### 方式一：交互模式（推荐）

```bash
cd "E:\MY AI\ValQ\license-manager"
node license-mgr.js
```

进入交互模式后，在 `>` 提示符后输入命令：

| 命令 | 说明 |
|------|------|
| `list` | 列出所有授权记录 |
| `query <机器码>` | 查询指定机器码的授权 |
| `gen <机器码> <类型> <有效期> [输出路径]` | 生成新授权 |
| `help` | 显示帮助信息 |
| `exit` | 退出 |

**交互模式示例：**

```
> list

共 2 条授权记录:

✅ 1. 88594C5EF7B300AAE81A8337FE014C2F | pro | 2036/05/27
✅ 2. A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6 | standard | 2027/05/27

> query 88594C5EF7B300AAE81A8337FE014C2F

找到 1 条授权记录:

1. 88594C5EF7B300AAE81A8337FE014C2F
   类型: pro
   状态: ✅ 有效
   到期: 2036/05/27 (剩余 3653 天)
   文件: 88594C5EF7B300AAE81A8337FE014C2F-pro-10y.lic

> gen 88594C5EF7B300AAE81A8337FE014C2F pro 10y license.lic
✅ AIDM 授权已生成: license.lic

> exit
```

### 方式二：命令行模式

```bash
# 列出所有授权
node license-mgr.js list

# 查询指定机器码
node license-mgr.js query 88594C5EF7B300AAE81A8337FE014C2F

# 生成授权
node license-mgr.js gen 88594C5EF7B300AAE81A8337FE014C2F pro 10y license.lic

# 显示帮助
node license-mgr.js help
```

## 🔄 完整工作流程

### 场景：用户申请授权

1. **用户提供机器码**
   - 用户打开 AIDM 软件
   - 点击底部黄色提示条上的"升级专业版"按钮
   - 在弹出的授权面板中复制机器码发送给管理员

2. **管理员生成授权文件**
   ```bash
   cd "E:\MY AI\ValQ\license-generator"
   node license-gen.js --machine <用户提供的机器码> --type pro --duration 1y
   ```

3. **发送授权文件给用户**
   - 将生成的 `.lic` 文件发送给用户（通过微信、邮件等）

4. **用户导入授权**
   - 用户打开 AIDM 软件
   - 点击"升级专业版" → "授权信息"
   - 点击"导入授权文件"按钮
   - 选择收到的 `.lic` 文件
   - 授权生效，所有功能解锁

## ⚠️ 注意事项

1. **密钥安全**：私钥 (`private.pem`) 是授权系统的核心，请妥善保管，切勿泄露。

2. **机器码唯一性**：授权文件与用户机器的硬件信息绑定，换电脑后需要重新生成授权。

3. **AES 密钥一致**：注册机、管理工具、Electron 客户端使用相同的 AES 密钥（`AIDM2026SecretKeyForAES256Encry!`，32字节），修改时需保持三端一致。

4. **有效期计算**：有效期从**生成授权文件当天**开始计算，不是从用户激活当天开始计算。

5. **试用版安全机制**：试用版授权使用 SHA-256 哈希完整性校验（基于机器码 + 到期时间 + 发放时间 + 固定盐值），手动修改 `.lic` 文件内容将导致解密失败，无法绕过试用限制。

## 🔒 安全加固机制

### 客户端防护

| 防护措施 | 说明 |
|---------|------|
| 禁用 DevTools | 生产模式下，F12 和 Ctrl+Shift+I 快捷键被禁用，用户无法直接打开浏览器开发者工具 |
| 禁用右键菜单 | 生产模式下，右键上下文菜单被禁用，防止查看页面源码 |
| 移除菜单栏 | 应用顶部不显示 File/Edit/View 等默认菜单 |

### 授权文件完整性校验

**试用版哈希校验**：
- 试用版授权不再使用简单的 `signature === 'trial'` 标记
- 而是基于 `机器码 + 到期时间 + 发放时间 + 固定盐值` 生成 SHA-256 哈希
- 任何手动修改 `.lic` 文件内容的行为都会导致 AES-GCM 解密失败或哈希校验不通过
- 盐值定义：`AIDM2026TrialIntegrityCheck`

**专业版 RSA 签名校验**：
- 专业版授权使用 RSA-2048 数字签名
- 签名数据包含：版本号、机器码、授权类型、到期时间、发放时间、功能列表
- 客户端嵌入公钥验证，只有持有对应私钥签发的授权文件才能通过验证

### AES-256-GCM 加密

- 所有授权文件内容均经过 AES-256-GCM 加密
- 密钥长度严格为 32 字节（`AIDM2026SecretKeyForAES256Encry!`）
- GCM 模式自带完整性校验（AuthTag），篡改密文将直接导致解密失败

## 🆘 常见问题

### Q: 忘记生成密钥对怎么办？
A: 运行 `node license-gen.js --generate-keys` 重新生成。但之前生成的授权文件将无法使用新密钥验证。

### Q: 授权文件生成后用户导入失败？
A: 检查：
1. AES 密钥是否三端一致
2. 公钥是否与客户端嵌入的公钥匹配
3. 机器码是否正确

### Q: 如何延长用户授权有效期？
A: 使用用户当前的机器码，重新生成一个有效期更长的授权文件，发给用户替换即可。
